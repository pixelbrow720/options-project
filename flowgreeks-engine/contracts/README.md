# Contracts — frontend bridge

This folder is the only thing that the frontend repo (`flowgreeks-frontend`)
depends on from this repo. Everything in `contracts/` is shaped for
consumption by a separate workspace.

## Layout

- `types/snapshot.ts` — canonical TypeScript types for the snapshot envelope,
  WS frames, and every payload sub-shape. Source of truth for both repos.
- `samples/` — real-shape JSON payloads. Use these for frontend dev when the
  backend isn't running.
  - `snapshot-spxw-friday.json` — full SPX/SPXW snapshot generated from
    Databento OPRA + GLBX pull on Friday 2026-05-22 close. Real spot, real
    GEX curve, real walls, real ATM IV. HIRO + flow are synthesized
    (cmbp-1 has no signed trades).
- `openapi.json` — REST spec auto-generated by FastAPI. Refresh with
  `scripts/export_contracts.sh`.
- `ws-frames.md` — WebSocket frame examples and reconnection semantics.

## Refresh workflow

When backend payload shapes change:

```bash
# from repo root
bash scripts/export_contracts.sh
```

That script will:
1. Boot the backend in test mode
2. Hit `/openapi.json` and write to `contracts/openapi.json`
3. Run a one-shot pipeline tick against fixtures and dump a snapshot
4. Diff `types/snapshot.ts` against the actual payload (manual reconcile if drift)

## What the frontend consumes

The frontend repo should treat this folder as read-only. To sync:

```bash
# in flowgreeks-frontend repo
npm run sync:contracts
# or manually:
cp -r ../flowgreeks-engine/contracts/types/* src/contracts/types/
cp -r ../flowgreeks-engine/contracts/samples/* src/contracts/samples/
cp ../flowgreeks-engine/contracts/openapi.json src/contracts/openapi.json
```

## REST endpoints (summary)

All `/v1/*` responses are wrapped: `{symbol, computed_at, next_update_in_seconds, data}`.

| Path                           | Returns                              |
| ------------------------------ | ------------------------------------ |
| `GET /v1/{symbol}/snapshot`    | `SnapshotEnvelope` (everything)      |
| `GET /v1/{symbol}/gex`         | `GexPayload` (OI-weighted)           |
| `GET /v1/{symbol}/walls`       | `WallsPayload`                       |
| `GET /v1/{symbol}/max-pain`    | `MaxPainPayload`                     |
| `GET /v1/{symbol}/iv`          | `IvPayload`                          |
| `GET /v1/{symbol}/0dte`        | `ZeroDtePayload`                     |
| `GET /v1/{symbol}/spot`        | `SpotPayload`                        |
| `GET /v1/{symbol}/futures-levels` | futures key levels                |
| `GET /v1/{symbol}/flow`        | `FlowPayload` (last 50 events)       |
| `GET /v1/{symbol}/hiro`        | `HiroPayload`                        |

## Streaming endpoints

| Path                                | Frames sent                          |
| ----------------------------------- | ------------------------------------ |
| `WS /v1/{symbol}/stream`            | `snapshot` (every pipeline tick) + `heartbeat` (25s)   |
| `WS /v1/{symbol}/stream/ticks`      | `tick` frames per spot/futures change                  |
| `GET /v1/{symbol}/stream/sse`       | Server-Sent Events fallback for `snapshot` frames      |

## Auth

Both REST and WS require `X-API-Key: ak_<token>` header (or `?key=<token>`
query param for WS). Rate limit 120 req/min per key on REST. WS connection
cap is 5 per key.

## Versioning

When breaking shape change required: bump `/v1/` to `/v2/` rather than
mutate existing payloads. Frontend repo gets explicit migration window.
